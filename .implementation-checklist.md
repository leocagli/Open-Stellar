# Skills Marketplace Implementation Checklist

## Acceptance Criteria Verification

### ✅ Skill Listing

- [x] **`POST /api/agents/[id]/skills`** — register a skill offering
  - File: `app/api/agents/[id]/skills/route.ts`
  - Accepts: `{ skillId, name, description, priceXlm, endpoint }`
  - Validates all required fields
  - Returns 201 on success with skill object
  - Returns 400 on validation errors

- [x] **`GET /api/agents/[id]/skills`** — list skills offered by an agent
  - File: `app/api/agents/[id]/skills/route.ts`
  - Returns array of skills for the agent
  - Returns empty array if no skills registered

- [x] **`GET /api/skills?q=<search>&maxPrice=<xlm>`** — search across all agent skills
  - File: `app/api/skills/route.ts`
  - Query parameter `q`: searches name, description, skillId (case-insensitive)
  - Query parameter `maxPrice`: filters by maximum price
  - Returns sorted results (by price ascending)
  - Returns count of results

### ✅ x402 Payment Flow

- [x] **Consumer calls `GET /api/agents/[id]/skills/[skillId]/invoke`**
  - File: `app/api/agents/[id]/skills/[skillId]/invoke/route.ts`
  - Returns x402 payment challenge (402 status)
  - Challenge includes: quoteId, paymentRef, amount, address, options
  - Service ID format: `skill:{agentId}:{skillId}`
  - Returns 404 if skill not found
  - Supports chain selection via query param

- [x] **Consumer pays → server verifies payment → calls agent's endpoint**
  - File: `app/api/agents/[id]/skills/[skillId]/invoke/route.ts` (POST handler)
  - Verifies payment using `settleX402()`
  - Rejects if payment verification fails (402 status)
  - Calls agent's skill endpoint on successful payment
  - Forwards consumer payload to agent endpoint
  - Includes headers: `X-Payment-Ref`, `X-Paid-By`, `X-Skill-Id`
  - Returns both receipt and agent's response

- [x] **Agent receives the net payment in their x402 revenue**
  - Uses existing x402 settlement infrastructure
  - Receipt stored via `settleX402()` which calls `saveX402Receipt()`
  - Revenue tracked in x402 receipt store
  - Receipt includes: agentId, service, amount, txHash, chain

### ✅ Storage

## Architecture

```
Consumer                   Marketplace API                Agent
   |                             |                           |
   |-- GET /api/skills?q=X ----->|                           |
   |<-- [skill list] ------------|                           |
   |                             |                           |
   |-- GET .../invoke ---------->|                           |
   |<-- 402 payment challenge ---|                           |
   |                             |                           |
   |-- make payment ------------->|                           |
   |                             |                           |
   |-- POST .../invoke --------->|                           |
   |    (txHash, paymentRef)     |                           |
   |                             |-- verify payment -------->|
   |                             |<-- payment OK ------------|
   |                             |                           |
   |                             |-- POST /skill/endpoint -->|
   |                             |   (payload + headers)     |
   |                             |<-- skill result ----------|
   |                             |                           |
   |<-- receipt + result --------|                           |
```

- Skills are uniquely identified by `(agentId, skillId)` tuple
- Payment verification uses existing `settleX402()` infrastructure
- Failed endpoint calls still consume payment (consumer paid for attempt)
- All routes use Next.js App Router conventions
- API logging integrated via `createApiRouteLogger()`
